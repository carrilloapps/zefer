# app/api — Server-Side Routes

Minimal server-side logic. Only one route, read-only.

## Routes

### `/api/author` (GET)
- Returns GitHub profile data for `carrilloapps`
- Fetches from `api.github.com/users/carrilloapps`
- Cached in memory for 1 hour
- Falls back to static data if GitHub API fails
- Fields: login, name, avatar, bio, company, location, blog, twitter, publicRepos, followers, profileUrl, createdAt

## Rules

- No API route should ever receive, store, or log user content, passphrases, or encryption keys
